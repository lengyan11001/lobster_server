# 速推（xskill）扣费与龙虾对齐说明

## 扣费时机（与官方 REST / Skill 一致）

- **创建任务**：速推 MCP 工具 **`generate`**（对应 REST `POST /api/v3/tasks/create`）返回里常见字段 **`price`**（积分），表示本次任务消耗。
- **查询结果**：工具 **`get_result`** 为轮询，**不应再按同一次任务重复扣费**。

龙虾 MCP 代理（`mcp/http_server.py`）行为：

- 仅对 **`upstream_tool == "generate"`** 的响应做「按返回解析金额并记入 `record-call`」的动态扣费。
- **`get_result`** 路径下动态扣费恒为 **0**（若能力在库里配置了 `unit_credits` 仍可走预扣，一般 `task.get_result` 为 0）。

## 失败退款

- **创建阶段失败**（上游 error 或 HTTP 错误）：若存在 **pre-deduct**，走原有 **refund**；动态扣费在失败时不会产生扣款。
- **异步任务终态失败**（`get_result` 返回 `failed` / `error` / `cancelled` 等）：龙虾在 **create 成功时已扣** 的金额会按 **`task_id`** 从进程内缓存取出，并调用 **`POST /capabilities/refund`** 退回用户余额（与「速推失败应退款」对齐）。
- **任务成功**：清除该 `task_id` 的缓存，不退款。

## 限制

- `task_id → 已扣积分` 缓存在 **MCP 进程内存**，进程重启后会丢失；极端情况下失败退款可能无法自动对齐（需人工调余额或依赖日志）。
- 若 `generate` 响应中无法解析 **`task_id`**，无法关联后续 `get_result`，失败时**不会**自动退款。
