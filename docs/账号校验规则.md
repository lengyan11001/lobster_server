# 账号校验规则（注册/登录）

服务端对「账号」的校验与查重规则，与 lobster_online 前端约定一致。

---

## 一、账号格式（注册与登录）

| 规则 | 说明 |
|------|------|
| 必填 | 不能为空，去除首尾空格后长度 ≥ 1 |
| 长度 | 2～64 个字符（去除首尾空格后） |
| 首字符 | 必须为字母（a-z, A-Z） |
| 字符集 | 仅允许：英文字母、数字、`.`、`_`、`-` |
| 规范化 | 存库与查重统一使用**小写** |

合法示例：`myname`、`User123`、`a.b`、`hello_world`。  
非法：空、`1user`（数字开头）、`ab`（可，但若不足 2 字符则非法）、`user@mail`（含 `@` 不允许）。

---

## 二、注册时服务端校验顺序

1. **验证码**：先校验 `captcha_id` + `captcha_answer`，错误或过期则 400，不继续。
2. **账号格式**：按上表校验，不通过则 400，提示对应错误（如「账号须字母开头」「账号须 2～64 位」「账号仅允许字母、数字、._-」）。
3. **密码**：长度 ≥ 6，否则 400「密码至少 6 位」。
4. **重复检查**：用规范化后的小写账号查 `User.email`（或唯一索引字段），若已存在则 400「该账号已注册」。
5. 通过后写入用户，返回 token。

---

## 三、登录时服务端

- 用请求中的账号去除首尾空格后转**小写**再查库，与注册存库一致，保证大小写不敏感、唯一性一致。

---

## 四、实现位置

- **lobster_server**：`backend/app/api/auth.py`  
  - `_normalize_and_validate_account()`：格式校验并返回小写账号。  
  - 注册：先验证码 → 账号格式 → 密码长度 → **查重（用户名不能重复）** → 写入。  
  - 登录：账号小写查库。

与 lobster_online 前端输入校验（如 `pattern`、`minlength`）保持一致，服务端为最终权威。
