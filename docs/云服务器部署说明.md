# 龙虾 (Lobster) 云服务器部署说明

在云服务器（Linux）上跑起在线版需要：**安装环境 → 部署包（lobster-server）→ 配置 .env → 启动后端与 MCP →（可选）反向代理与开机自启**。

**目录约定**

- **`lobster_online/`**：本机完整包，本地开发、打 online 版、改功能通常在此目录（本工作区**无**顶层 `lobster/` 目录）。
- **`lobster-server/`**：仅服务器部署用；由你们从 `lobster_online` 同步或打包得到（若有 `scripts/build_server_package.sh` 等，在对应仓库根执行）；云服 Git 拉取后只使用 **`lobster-server/`** 启动服务。

更新部署包：在开发机按仓库内**实际脚本**执行（若有），再提交并推送 `lobster-server/`。

---

## 一、服务器上要跑什么

| 进程 | 端口 | 说明 |
|------|------|------|
| **Backend** | 8000 | FastAPI：仅提供 API（登录、对话、计费、发布等），不提供网页；用户通过 **lobster_online** 本地前端访问本服务。 |
| **MCP** | 8001（仅本机） | 能力网关：速推/技能/发布等工具，只供 Backend 调用，不对外暴露 |

| **OpenClaw Gateway** | 18789（可选） | 若用 OpenClaw 做对话路由才需要，否则对话可走直连模型 |

**本地/服务器仅 IP 无域名**：不配 `PUBLIC_BASE_URL` 时，自动用**本机 LAN IP:PORT** 作为回调根地址（本地直连或单机可用）。服务器只有公网 IP 时在 `.env` 填 `PUBLIC_BASE_URL=http://公网IP:8000`；有域名则填 `https://域名`。**企业微信接收消息回调**也依赖此配置，详见 [企业微信服务器部署](wecom_server_deploy.md)。本地盒子无公网时，前端 API 指向该服务端即可，扫码后微信回调到服务端，登录成功页展示 Token 供复制。

对外只需开放 **8000**（或由 Nginx 反代后的 80/443）。8001、18789 仅本机访问。

**登录/注册与验证码传输加密**：账号密码登录、注册及图片验证码答案会经网络提交，生产环境**必须使用 HTTPS**（Nginx 配置 SSL 或前置 TLS 终止），以保证密码与验证码在传输过程中加密，避免被窃听或篡改。

## 二、环境要求

- **操作系统**：Linux（如 Ubuntu 22.04、CentOS 7+）
- **Python**：3.10 或 3.12，且能装 pip 依赖
- **Node.js**（可选）：仅在使用 OpenClaw Gateway 时需要
- **Chromium/Playwright**（可选）：仅在使用「发布到抖音」等浏览器能力时需要

```bash
# Ubuntu/Debian 示例
sudo apt update
sudo apt install -y python3.10 python3.10-venv python3-pip

# 若需要发布到抖音（无头浏览器）
# sudo apt install -y libnss3 libnspr4 libatk1.0-0 libatk-bridge2.0-0 libcups2 libdrm2 libxkbcommon0 libxcomposite1 libxdamage1 libxfixes3 libxrandr2 libgbm1 libasound2
# 安装后进入项目目录执行: python3 -m playwright install chromium
```

## 三、部署步骤

### 0. Git 克隆后快速启动（推荐）

代码已推送到 Git 仓库（且含 `lobster-server/`）时，在服务器上**进入部署包目录**执行：

```bash
git clone <你的仓库地址> /opt/repo
cd /opt/repo/lobster-server
chmod +x scripts/*.sh
./scripts/server_install.sh
```

- 若没有 `.env`，脚本会复制 `.env.example` 并提示你先编辑再启动。
- 编辑 `.env` 填写 `SECRET_KEY`、`SUTUI_SERVER_TOKEN` 等（见下文「3. 配置 .env」）。

然后一键启动 MCP + Backend：

```bash
./scripts/server_start.sh
```

- 会先检测 8001 是否已有 MCP，无则后台启动 MCP，再前台启动 Backend（端口 8000）。
- 访问：`http://服务器IP:8000`。生产环境建议改用 systemd（见下文「方式 B」）。

**日常更新代码并重启**（在部署目录执行）：

```bash
cd /opt/lobster-server   # 或你的部署根目录
bash scripts/server_update_and_restart.sh
```

会拉取 `origin main` 并重启 `lobster-backend`、`lobster-mcp`（若已用 systemd）。本地开发机可配置 `LOBSTER_DEPLOY_HOST=user@服务器IP` 后执行 `bash scripts/deploy_from_local.sh` 一键完成**推送后的远程拉取与重启**（就这一条链路，无需其它部署脚本）。**lobster_online** 为本地前端，不部署到本服务器；本机预览见 `lobster_online/docs/本地启动与重启.md`。

### 1. 上传代码（非 Git 时）

将 **`lobster-server/`** 整个目录上传到服务器（例如 `/opt/lobster-server`），保证目录内包含：

- `backend/`、`mcp/`、`publisher/`、`openclaw/`（至少 `openclaw.json`）
- `scripts/`（含 `server_install.sh`、`server_start.sh`）
- `requirements.txt`、`.env.example`、`skill_registry.json`

### 2. 安装 Python 依赖（未用 server_install.sh 时）

```bash
cd /opt/lobster-server
python3 -m venv .venv
source .venv/bin/activate   # Linux/Mac
pip install -r requirements.txt
```

### 3. 配置 .env

```bash
cd /opt/lobster-server
cp .env.example .env
vim .env   # 或 nano .env
```

**必填 / 强烈建议：**

| 变量 | 说明 |
|------|------|
| `SECRET_KEY` | 随机长字符串，用于 JWT 等，生产环境必改 |
| `SUTUI_SERVER_TOKEN` | 速推 API Token，能力由服务器转发时必填 |
| （已废弃 `SERVE_FRONTEND`） | 本构建仅 API，前端由 lobster_online 提供，无需配置 |
| `HOST` | 填 `0.0.0.0`，以便外网访问 |
| `PORT` | 默认 8000 |
| `LOBSTER_RECHARGE_ADMIN_SECRET` | 完成充值/技能解锁订单时用的密钥，仅服务端知晓 |

**按需填写：**

- `PUBLIC_BASE_URL`：图生视频等需被速推拉图时，填公网访问地址（如 `https://your-domain.com`）
- `DEBUG`：生产建议 `false`
- `LOG_LEVEL`：生产可设 `info` 减少日志

**一键配置（可选）**：脚本在 **lobster_server** 仓库的 `scripts/config_env.py`。在服务器项目根目录执行 `python3 scripts/config_env.py` 可交互式填写微信登录/支付、速推 Token、回调地址等；或先 `export CONFIG_WECHAT_APP_ID=xxx ...` 再运行，将把 CONFIG_* 写入 `.env`。

**MCP 读到的环境**：Backend 和 MCP 都会读部署根目录（即 `lobster-server` 根目录）下的 `.env`；若用 systemd，工作目录设为该根目录，或把 `SUTUI_SERVER_TOKEN` 等写进 systemd 的 `Environment=`。

### 4. 启动方式

**方式 A：前台直接跑（适合先试跑）**

```bash
cd /opt/lobster-server
source .venv/bin/activate
export PYTHONPATH=/opt/lobster-server

# 终端 1：MCP（必须先起，否则 Backend 会尝试自启一次）
python3 -m mcp --port 8001

# 终端 2：Backend
python3 -m backend.run
```

然后访问 `http://服务器IP:8000`。

**方式 B：用 systemd 常驻（推荐生产）**

以下路径均以部署根目录 `/opt/lobster-server` 为例。推荐直接运行仓库内脚本安装/更新 systemd：

```bash
cd /opt/lobster-server
bash scripts/install_systemd_units.sh /opt/lobster-server
sudo systemctl daemon-reload
sudo systemctl enable lobster-mcp lobster-backend lobster-background
sudo systemctl restart lobster-mcp lobster-backend lobster-background
```

`lobster-backend` 默认 `BACKEND_WORKERS=2`，只负责 Web API；`lobster-background` 单进程负责速推 LLM 探测、速推对账、Meta 定时发布，避免多 worker 后后台任务重复跑。

如需手工建 service，可参考以下三个文件。

`/etc/systemd/system/lobster-mcp.service`：

```ini
[Unit]
Description=Lobster MCP Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/lobster-server
Environment=PYTHONPATH=/opt/lobster-server
EnvironmentFile=/opt/lobster-server/.env
ExecStart=/opt/lobster-server/.venv/bin/python3 -m mcp --port 8001
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

`/etc/systemd/system/lobster-backend.service`：

```ini
[Unit]
Description=Lobster Backend
After=network.target lobster-mcp.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/lobster-server
Environment=PYTHONPATH=/opt/lobster-server
Environment=BACKEND_WORKERS=2
Environment=LOBSTER_BACKEND_AUTOSTART_MCP=0
EnvironmentFile=/opt/lobster-server/.env
ExecStart=/opt/lobster-server/.venv/bin/python3 -m backend.run
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

`/etc/systemd/system/lobster-background.service`：

```ini
[Unit]
Description=Lobster Background Worker
After=network.target postgresql.service lobster-mcp.service lobster-backend.service
Wants=lobster-mcp.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/lobster-server
Environment=PYTHONPATH=/opt/lobster-server
EnvironmentFile=/opt/lobster-server/.env
ExecStart=/opt/lobster-server/.venv/bin/python3 -m backend.background_worker
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable lobster-mcp lobster-backend lobster-background
sudo systemctl start lobster-mcp lobster-backend lobster-background
sudo systemctl status lobster-mcp lobster-backend lobster-background
```

日志：

```bash
journalctl -u lobster-backend -f
journalctl -u lobster-mcp -f
journalctl -u lobster-background -f
```

应用文件日志按日期写入部署目录：

```text
logs/app-YYYY-MM-DD.log
logs/background-YYYY-MM-DD.log
logs/h5-YYYY-MM-DD.log
```

默认只保留 3 天内日志；可用环境变量调整：

```env
LOBSTER_LOG_RETENTION_DAYS=3
LOBSTER_DIAGNOSTICS_RETENTION_DAYS=3
```

`/api/logs` 默认读取当天 `app-YYYY-MM-DD.log` 的末尾内容。旧版 `logs/app.log` 会在启动时迁移到日期文件；为兼容部分脚本，支持系统会尽量创建 `logs/app.log` 指向当天日志的软链接。

用户上传的诊断包位于 `diagnostics_uploads/user_<id>/diag_*`，上传和列表查询时都会清理 3 天前的诊断目录。部署脚本也会执行一次 `journalctl --vacuum-time=3d`，避免 systemd journal 长期占盘。

### 多 worker 与后台任务拆分

生产环境启用多 worker 后，必须遵守下面的进程边界：

| 进程 | systemd unit | 端口 | 职责 |
|------|--------------|------|------|
| Web API | `lobster-backend.service` | `8000` | FastAPI 接口；当前默认 `BACKEND_WORKERS=2`。 |
| MCP | `lobster-mcp.service` | `8001` | 能力工具服务。 |
| H5 | `lobster-h5.service` | `8010` | H5 静态/会话入口。 |
| 后台循环 | `lobster-background.service` | 无 | 速推 LLM 探测、速推对账、Meta 定时发布等单实例循环。 |

不要把后台循环重新放回 FastAPI lifespan。多 worker 下 lifespan 会在每个 worker 内执行，定时探测、对账、定时发布会重复跑。

`lobster-backend.service` 里必须保留：

```ini
Environment=BACKEND_WORKERS=2
Environment=LOBSTER_BACKEND_AUTOSTART_MCP=0
```

`BACKEND_WORKERS=2` 是当前 2 核 3.6GiB 服务器的起步配置。升到 4 worker 前至少确认：

- CPU load 长时间接近或超过核心数。
- 单 worker CPU 已经打满，而 DB、外部 API、网络不是瓶颈。
- PostgreSQL 连接数仍有余量。
- 内存没有明显 swap 抖动。

数据库连接池按多 worker 默认：

```env
DB_POOL_SIZE=5
DB_MAX_OVERFLOW=10
```

如果生产 `.env` 没显式配置这两个值，就使用代码默认值。若未来增加 worker，需要同步评估 `worker 数 * (pool_size + max_overflow)` 对 PostgreSQL 最大连接数的影响。

启动期建表、迁移、能力目录同步已经使用 PostgreSQL advisory lock 串行化，避免多个 worker 同时做 schema/catalog 初始化。

常用巡检命令：

```bash
systemctl is-active lobster-backend lobster-mcp lobster-background lobster-h5 postgresql
pgrep -af 'python3 -m backend.run|python3 -m backend.background_worker|python3 -m mcp|python3 -m backend.h5_run'
ss -ltnp | grep -E ':(8000|8001|8010|5432)\b'
curl -fsS http://127.0.0.1:8000/api/health
sudo -u postgres psql -d lobster -Atc "select concat('total=', count(*)), concat('active=', count(*) filter (where state='active')), concat('idle=', count(*) filter (where state='idle')) from pg_stat_activity where datname='lobster';"
```

## 四、可选：Nginx 反代 + HTTPS

对外只暴露 80/443，由 Nginx 转发到本机 8000：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

HTTPS 可用 `certbot` 申请证书并改 `listen 443 ssl` 等。

**.env 中**：若用户通过 `https://your-domain.com` 访问，则 `PUBLIC_BASE_URL=https://your-domain.com`（图生视频等会用到）。

**宝塔 / 多 location 注意**：若 `server` 里除 `/api/`、`/static/` 等指向 8000 外，还有 **`location ^~ /` 指向其它端口**（如 8003），则必须**单独**增加一段，把 **`/client/`** 反代到 **8000**，否则在线客户端的 `CLIENT_CODE_MANIFEST_URL`（`/client/client-code/manifest.json`）会落到错误上游并出现 **502**。仓库内参考片段：`scripts/nginx_extension_api_51ins_client.conf`；一键写入大陆机 extension 并重载：`bash scripts/push_nginx_client_extension.sh`（依赖 `.env.deploy`）。

## 五、可选：OpenClaw Gateway

若希望对话走 OpenClaw（多模型路由），需再起 OpenClaw Gateway（Node）：

- 安装 Node.js，在部署根目录（如 `/opt/lobster-server`）执行 `npm install openclaw`（或使用已有 nodejs 目录）
- 启动：`OPENCLAW_CONFIG_PATH=/opt/lobster-server/openclaw/openclaw.json OPENCLAW_STATE_DIR=/opt/lobster-server/openclaw node node_modules/openclaw/openclaw.mjs gateway --port 18789`
- `.env` 中配置 `OPENCLAW_GATEWAY_URL=http://127.0.0.1:18789` 和 `OPENCLAW_GATEWAY_TOKEN`

不配 OpenClaw 时，对话仍可用直连模型（在 openclaw/.env 或系统配置里填各模型 API Key）。

## 六、可选：发布到抖音（Playwright Chromium）

发布到抖音依赖无头浏览器，需安装 Playwright 的 Chromium：

```bash
cd /opt/lobster-server
source .venv/bin/activate
python3 -m playwright install chromium
```

若报缺系统库，按 Playwright 文档安装对应依赖（如 `libnss3`、`libatk-bridge2.0-0` 等）。

## 八、常见问题

- **能力调用报「Token 未配置」**：检查 `.env` 中 `SUTUI_SERVER_TOKEN` 是否填写，且 MCP 进程能读到该环境变量（同一 `.env` 或 systemd 的 EnvironmentFile）。
- **图生视频报 Failed to download**：速推要能访问素材 URL，请把 `PUBLIC_BASE_URL` 设为公网可访问的根地址（如 `https://your-domain.com`）。
- **发布到抖音失败**：确认已 `playwright install chromium`，且运行用户有写 `browser_data` 的权限。

按上述步骤即可在云服务器上跑起龙虾在线版。
