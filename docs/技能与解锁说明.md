# 技能列表与解锁说明

- 能力配置与展示以 **server 为准**；`skill_registry.json` 中配置技能包，支持 `show_in_store`、`default_installed`、`unlock_price_credits` 等。
- 用户解锁状态存 **服务器**（SkillUnlock 表），以服务器为准。权限校验可能发生在 **会话**（如发布、企微回复）或 **其他界面**（如发布管理添加账号、企微配置页）。

## 当前 4 个能力

| 能力 | 技能包 ID | 展示与解锁 |
|------|-----------|------------|
| 速推 | sutui_mcp | 默认已安装、直接可用（在线版由速推登录提供，卡片显示「已安装」） |
| 发抖音 | douyin_publish | 默认已安装、直接可用；未解锁「添加更多发布账号」时仅可添加 1 个账号 |
| 发小红书 | xiaohongshu_publish | 同上 |
| 企业微信自动回复 | wecom_reply | 技能商店中 1000 积分解锁；未解锁不能进入配置页，解锁后正常使用 |

- **添加更多发布账号**（add_more_publish_accounts）：1000 积分解锁，**不在技能商店展示**（`show_in_store: false`）。在发布管理中添加第 2 个账号时，若未解锁则接口返回 402，前端可引导用户调用 `POST /skills/unlock-by-credits` 解锁后再添加。

## 技能包字段（skill_registry.json）

- `default_installed`: true 表示默认已安装、直接可用，商店显示「已安装」，不展示积分解锁。
- `show_in_store`: false 表示不在技能商店列表中展示，仅用于接口校验（如添加更多账号时解锁）。
- `unlock_price_credits`: 积分解锁金额；需解锁时由前端/其他界面引导调用 `POST /skills/unlock-by-credits`。

## 后续扩展

- 能力可分为：**本地预装**（当前 4 个）、**需在线下载**（后续由 server 下载 skill 包并执行安装脚本）。
- 可增加其他解锁或收费模式，由 server 配置与 SkillUnlock/订单逻辑扩展。
