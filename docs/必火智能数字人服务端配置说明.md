# 必火智能数字人服务端配置说明

本文档用于部署和排查必火智能数字人功能。该功能前端入口在 `lobster_online`，核心业务接口在 `lobster_server`，上游能力由 HiFly 接口提供，平台侧负责统一托管 Token、素材入库、任务状态同步和算力计费。

## 一、整体调用链路

1. 用户在客户端进入必火智能数字人工作台。
2. 前端请求 `lobster_server` 的 `/api/hifly/my/*` 接口。
3. `lobster_server` 使用服务端配置的 `HIFLY_DEFAULT_TOKEN` 调用 HiFly 上游接口。
4. 创建任务时，服务端先调用平台计费接口预扣算力。
5. 任务完成后，服务端根据实际结果进行最终结算或失败退款。
6. 数字人、声音、视频任务记录保存在服务端数据库中。
7. 声音试听优先使用 HiFly 返回的 `demo_url`，没有时回退到用户上传的样本音频，并通过服务端代理地址播放。

## 二、必须配置的环境变量

配置文件通常位于：

```text
lobster_server/.env
```

### 1. 基础运行配置

```env
HOST=127.0.0.1
PORT=8002
DATABASE_URL=sqlite:///./lobster.db
SECRET_KEY=请替换为稳定且安全的随机字符串
LOBSTER_INDEPENDENT_AUTH=true
```

注意：

- `SECRET_KEY` 不要频繁更换，否则已有 JWT、签名素材链接、声音试听代理链接可能失效。
- `DATABASE_URL` 指向的数据库必须持久化，不能部署到会被清理的临时目录。

### 2. HiFly 上游 Token

```env
HIFLY_DEFAULT_TOKEN=你的HiFly服务端Token
```

注意：

- 该 Token 只能配置在服务端，不能暴露给客户端。
- 前端不应该让用户填写 HiFly Token。
- 如果不配置，数字人、声音、视频创建和公共资源拉取会失败。

### 3. 算力计费配置

```env
AUTH_SERVER_BASE=http://127.0.0.1:8002
LOBSTER_MCP_BILLING_INTERNAL_KEY=请替换为一个服务端内部密钥
```

本地单体部署时，`lobster_server` 自己就是认证和积分服务，因此：

```env
AUTH_SERVER_BASE=http://127.0.0.1:8002
```

如果是多机部署，`AUTH_SERVER_BASE` 应该填写认证中心公网或内网可访问地址，例如：

```env
AUTH_SERVER_BASE=https://api.example.com
```

注意：

- `AUTH_SERVER_BASE` 必须能访问 `/capabilities/pre-deduct`、`/capabilities/record-call`、`/capabilities/refund`。
- `LOBSTER_MCP_BILLING_INTERNAL_KEY` 要和认证中心服务端配置保持一致。
- 如果缺少 `AUTH_SERVER_BASE`，创建视频时会报：`未配置 AUTH_SERVER_BASE，无法完成 HiFly 算力计费`。
- 如果密钥不一致，可能出现预扣、结算或退款接口鉴权失败。

### 4. 素材和试听音频访问配置

```env
PUBLIC_BASE_URL=https://你的服务端公网域名
LAN_PUBLIC_BASE_URL=http://局域网IP:8002
```

用途：

- 生成素材签名 URL。
- 生成声音试听代理 URL。
- 让浏览器或外部服务能够访问上传的音频、图片、视频。

本地开发可不填，服务会尝试使用请求来源生成地址。但正式部署建议填写 `PUBLIC_BASE_URL`。

注意：

- 正式环境不要配置成 `127.0.0.1`，否则用户浏览器或外部服务无法访问。
- 如果前端和服务端不在同一台机器，`PUBLIC_BASE_URL` 必须是前端用户可访问的地址。
- 声音试听失败时，优先检查返回的 `demo_url` 是否能直接在浏览器打开。

## 三、数据库相关表

必火智能数字人相关数据主要保存在以下表：

```text
user_hifly_avatar_assets
user_hifly_voice_assets
user_hifly_video_assets
assets
```

### 1. 声音表 `user_hifly_voice_assets`

关键字段：

- `hifly_task_id`：HiFly 声音克隆任务 ID。
- `hifly_voice_id`：任务成功后返回的声音 ID。
- `demo_url`：HiFly 返回的试听音频地址。
- `status`：任务状态，常见值为 `processing`、`success`、`failed`。
- `meta`：保存原始响应和上传音频信息。

`meta.upload_meta` 中应包含：

```json
{
  "file_id": "上游文件ID",
  "content_type": "audio/mpeg",
  "filename": "上传文件名",
  "size": 123456,
  "extension": "mp3",
  "source_asset_id": "本地素材ID",
  "source_url": "本地或对象存储地址"
}
```

如果 `demo_url` 为空，服务端会尝试使用：

1. `meta.task_raw` 里的 `demo_url`、`audio_url`、`preview_url`。
2. `meta.create_raw` 里的 `demo_url`、`audio_url`、`preview_url`。
3. `meta.upload_meta.source_url`。
4. `meta.upload_meta.source_asset_id` 生成 `/api/assets/file/{asset_id}` 签名链接。
5. 最终返回 `/api/hifly/my/voice/{id}/preview` 代理试听地址。

### 2. 素材表 `assets`

用户上传的声音样本、数字人图片、数字人视频会保存到素材表。

关键字段：

- `asset_id`：素材唯一 ID。
- `user_id`：所属用户。
- `filename`：本地文件名或对象存储 key。
- `media_type`：`audio`、`image`、`video`。
- `source_url`：公网地址或原始来源地址。

如果声音试听按钮存在但播放失败，需要检查 `assets` 中对应 `asset_id` 的文件是否真实存在，`source_url` 是否可访问。

## 四、计费规则和状态

当前视频创作接入平台算力计费：

- 创建数字人口播任务时先预扣。
- 任务完成后按实际时长结算。
- 任务失败时退款。

计费信息通常保存在视频任务记录的 `meta.billing` 中。

排查重点：

- 创建任务响应是否包含 `billing`。
- 轮询任务响应是否包含顶层 `billing`。
- 任务失败后是否调用退款。
- 实际完成时是否按 `duration` 结算。

如果报算力相关错误，先检查：

```env
AUTH_SERVER_BASE
LOBSTER_MCP_BILLING_INTERNAL_KEY
SECRET_KEY
```

然后确认当前用户 JWT 是否正常，认证中心是否能识别该用户。

## 五、服务启动和端口

本地常用启动方式：

```powershell
python backend\run.py
```

服务端默认端口：

```text
http://127.0.0.1:8002
```

客户端通常通过 URL 参数指定服务端：

```text
http://127.0.0.1:8000/?api=http://127.0.0.1:8002&local_api=http://127.0.0.1:8000
```

部署时要确保：

- 客户端配置的 `api` 指向正确的 `lobster_server`。
- 服务端 CORS 配置允许客户端来源访问。
- 反向代理不要拦截 `Authorization` 请求头。
- 上传文件大小限制满足音频、视频上传需求。

## 六、常见问题排查

### 1. 报 `未配置 AUTH_SERVER_BASE，无法完成 HiFly 算力计费`

原因：

- `.env` 没有配置 `AUTH_SERVER_BASE`。
- 配置类没有声明 `auth_server_base` 字段。
- 服务端修改 `.env` 或代码后没有重启。

处理：

1. 确认 `.env` 有：

```env
AUTH_SERVER_BASE=http://127.0.0.1:8002
```

2. 确认 `backend/app/core/config.py` 的 `Settings` 有：

```python
auth_server_base: Optional[str] = None
```

3. 重启 `lobster_server`。

### 2. 声音创建成功，但“我的声音”无法试听

重点检查：

- `user_hifly_voice_assets.demo_url` 是否为空。
- `user_hifly_voice_assets.meta.upload_meta.source_asset_id` 是否存在。
- `assets` 表里是否存在对应 `asset_id`。
- 素材文件是否真实保存成功。
- 返回给前端的 `demo_url` 是否是 `/api/hifly/my/voice/{id}/preview?...`。
- 直接打开该 `demo_url` 是否返回音频内容，而不是 401、403、404 或 HTML 错误页。

可能原因：

- HiFly 上游没有返回试听地址。
- 上传样本音频没有成功入库到 `assets`。
- `PUBLIC_BASE_URL` 配成了外部不可访问的地址。
- `SECRET_KEY` 变更导致试听代理签名失效。
- 用户 JWT 过期或请求试听接口时没有鉴权。

### 3. 创建任务时提示 HiFly Token 相关错误

检查：

```env
HIFLY_DEFAULT_TOKEN=...
```

注意：

- Token 必须放服务端。
- 修改后必须重启服务端。
- 如果 Token 过期，需要从 HiFly 重新获取。

### 4. 预扣成功但完成后没有结算

检查：

- 前端是否持续轮询任务。
- 服务端 `/api/hifly/my/video/task` 是否返回任务终态。
- HiFly 结果里是否有 `duration`。
- `meta.billing` 是否已经标记结算，避免重复扣费。
- 日志里是否有 `record-call` 或 `refund` 失败。

### 5. 页面仍显示旧文案或旧逻辑

处理：

- 确认客户端 `static/index.html` 引用的 `hifly-digital-human.js?v=...` 已更新。
- 浏览器执行强制刷新。
- 确认访问的是正确的 8000 客户端，而不是旧部署。

## 七、上线前检查清单

上线前建议逐项确认：

- [ ] `HIFLY_DEFAULT_TOKEN` 已配置且有效。
- [ ] `AUTH_SERVER_BASE` 已配置且服务端可访问。
- [ ] `LOBSTER_MCP_BILLING_INTERNAL_KEY` 与认证中心一致。
- [ ] `SECRET_KEY` 稳定且不会每次部署变化。
- [ ] 数据库路径是持久化路径。
- [ ] `PUBLIC_BASE_URL` 是用户浏览器可访问地址。
- [ ] 创建声音后，“我的声音”可以试听。
- [ ] 创建视频时可以预扣算力。
- [ ] 视频完成后能按实际时长结算。
- [ ] 视频失败时能退款。
- [ ] 前端页面不再展示 HiFly 或飞影品牌字样。
- [ ] 前端不再要求用户输入 HiFly Token。

## 八、建议保留的日志关键词

排查时建议搜索这些关键词：

```text
hifly
pre-deduct
record-call
refund
billing
voice preview
asset
AUTH_SERVER_BASE
HIFLY_DEFAULT_TOKEN
```

如果部署人员无法判断问题属于前端还是服务端，优先查看浏览器 Network 中 `/api/hifly/my/*` 接口响应；只要响应体里出现上述服务端错误文案，通常就是 `lobster_server` 配置或上游接口问题。
