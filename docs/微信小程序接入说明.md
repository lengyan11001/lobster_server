# 微信小程序接入说明

小程序源码位于 `miniprogram/`，目标是让已开通 online 的用户在手机端绑定手机号，并下载保存 online 生成的素材。

## 当前开发进度

## 前置条件

1. 微信公众平台注册小程序，拿到 AppID 与 AppSecret。
2. 小程序后台开通手机号快速验证组件。
3. 小程序后台配置合法域名：
   - request 合法域名：`https://bhzn.top`
   - uploadFile 合法域名：`https://bhzn.top`
   - downloadFile 合法域名：`https://bhzn.top`
   - downloadFile 合法域名：`https://cdn-video.51sux.com`（当前火山 TOS/CDN 素材域名）
   - 如果后续安装包或素材换成新的 OSS/TOS/CDN 域名，也要把新域名加入 downloadFile 合法域名。
4. 服务器 `.env` 配置：

```env
WECHAT_APP_ID=wxbbf4a4f9a0fdd872
WECHAT_APP_SECRET=小程序AppSecret
WECHAT_MINIPROGRAM_PAGE=pages/index/index
PUBLIC_BASE_URL=https://bhzn.top
```

改完后重启 `lobster_server`。

`WECHAT_MINIPROGRAM_PAGE` 不是在微信公众平台后台单独填写的页面地址，它必须和小程序代码 `miniprogram/app.json` 里的页面路径一致。当前首页已注册为 `pages/index/index`。

## 小程序开发配置

1. 用微信开发者工具打开 `lobster_server/miniprogram`。
2. `project.config.json` 已配置正式 AppID：`wxbbf4a4f9a0fdd872`。如需本机调试其他小程序，可创建本机私有配置 `project.private.config.json`。
3. 如服务器域名不是 `https://bhzn.top`，修改 `miniprogram/config.js`：

```js
module.exports = {
  API_BASE: "https://你的服务器域名",
  APP_NAME: "必火AI员工"
};
```

## 用户隐私保护指引填写

提交审核前，在微信公众平台「设置」/「用户隐私保护指引」里只保留当前小程序实际使用的信息类型，并补全用途。当前代码实际使用如下：

| 信息类型 | 是否使用 | 用途填写建议 |
|------|------|------|
| 手机号 | 使用 | 用于验证并绑定你在必火 AI 员工平台已开通的 online 账号，将手机端与电脑端设备关联，便于下发任务、查看消息和获取可下载素材。 |
| 相册（仅写入）权限 | 使用 | 用于将你在必火 AI 员工中生成或收到的图片、视频素材保存到手机相册，便于本地查看、下载和分享。 |
| 剪切板（写入） | 使用 | 用于在素材保存失败或用户主动复制时，将图片、视频素材链接复制到剪切板，便于在浏览器或其他应用中打开、下载。 |
| 选中的照片或视频信息 | 使用 | 用于在消息中选择并上传图片，作为图生视频、图片理解和素材生成的参考图。 |
| 选中的文件 | 未使用 | 当前小程序不选择或上传本地文件，后台不要勾选该项。 |

注意：

- 当前仅支持在消息里上传图片，使用 `wx.chooseMedia` 选择图片；暂不上传本地视频或文件。
- 当前 `miniprogram/app.json` 只声明了 `scope.writePhotosAlbum`，文案为「用于将你生成或收到的图片、视频素材保存到手机相册，便于本地查看、下载和分享」。
- 微信后台如果提示某项「用途未填写」，优先检查后台隐私指引是否勾选了未使用的信息类型；未使用的项目直接取消勾选。

体验成员访问开发版/体验版时，不是在小程序后台配置 `pages/index/index`。操作路径是：

1. 微信开发者工具打开 `lobster_server/miniprogram`。
2. 右上角“上传”，填写版本号和备注。
3. 微信公众平台 -> 版本管理 -> 开发版本，设置为体验版。
4. 微信公众平台 -> 成员管理 -> 体验成员，确认测试人员微信号已加入。
5. 分享“体验版二维码”给体验成员；不要分享开发者工具本机预览码给普通体验成员。

## 业务流程

1. 小程序启动后调用 `wx.login`，请求 `POST /api/mobile/wechat-login`。
2. 如果微信 openid 已经绑定到平台手机号账号，服务端直接返回手机号账号 token。
3. 如果还没绑定，用户点击“授权手机号”，小程序拿到微信手机号 `code`，请求 `POST /api/mobile/devices/bind`。
4. 服务端校验手机号是否已在平台内注册：
   - 已注册：绑定手机设备与 online 账号，返回账号 token。
   - 未注册：返回“该手机号未在平台内注册过，没有 online 版本”。
5. 素材页请求 `GET /api/mobile/downloads?device_id=...`，返回可预览和可下载链接。
6. 小程序优先通过 `wx.downloadFile` 直连下载 HTTPS 素材链接，再调用 `wx.saveImageToPhotosAlbum` 或 `wx.saveVideoToPhotosAlbum` 保存到相册；直链失败时再回退到 `bhzn.top` 下载代理。

## 素材下载新方式

小程序承担“保存到相册”的主路径，H5 只保留预览、复制链接和外部浏览器下载提示。原因是普通 H5 在微信内和 iOS Safari 中无法稳定直接写入系统相册。

### 服务端返回

`GET /api/mobile/downloads?device_id=...` 会汇总三类来源：

1. 素材库 `assets.source_url`
2. 定时任务完成结果中的媒体链接
3. H5 会话结果中的媒体链接

每条记录返回：

| 字段 | 说明 |
|------|------|
| `url` | 原始公网 URL。 |
| `preview_url` | 优先为 HTTPS 原始公网 URL，用于小程序直连预览。 |
| `download_url` | 优先为 HTTPS 原始公网 URL，用于小程序直连保存相册。 |
| `proxy_preview_url` | `/api/h5-chat/media` 预览代理兜底链接。 |
| `proxy_download_url` | `/api/h5-chat/media` 下载代理兜底链接。 |
| `media_type` | `image` / `video` / `audio` / `media`。 |
| `title` | 展示标题。 |
| `created_at` | 创建时间。 |

### 小程序保存

小程序端逻辑位于 `miniprogram/utils/media.js`：

1. 优先使用 `download_url`，也就是 TOS/CDN 直链。
2. 调用 `wx.downloadFile` 下载到小程序本地临时/用户目录。
3. 图片调用 `wx.saveImageToPhotosAlbum`。
4. 视频调用 `wx.saveVideoToPhotosAlbum`。
5. 音频或未知类型默认复制链接。
6. 用户拒绝相册权限时，引导 `wx.openSetting`。
7. 直链下载失败时自动尝试 `proxy_download_url`；仍失败时复制原始链接，避免用户拿不到素材。

## 安装包放 CDN/OSS/TOS

online 客户端更新只认 manifest 里的 `bundle_url`，这个地址可以是任意 HTTPS CDN/OSS/TOS 直链。推荐做法：

1. 先用 `lobster_online/scripts/pack_client_code_ota.py` 打 OTA zip。
2. 把 zip 上传到 TOS/OSS/CDN，例如当前可用 `https://cdn-video.51sux.com/client-code/...zip`。
3. 发布 manifest 时让 `bundle_url` 指向 CDN 直链：

```bash
python scripts/publish_client_code_ota_to_server.py D:\lobster_online_client_code_ota_xxx.zip --version 1.0.xx --build xx --bundle-url https://cdn-video.51sux.com/client-code/lobster_online_client_code_ota_xxx.zip
```

这样 manifest 仍从 `https://bhzn.top/client/client-code/manifest.json` 下发，但真正下载大包走 CDN。

### 生成侧要求

online 生成后的素材应优先保存稳定 HTTPS 公网链接：

1. 优先走火山 TOS。
2. TOS 配置由云端 `custom_configs.json` 维护。
3. online 本机缺少 TOS 配置时，通过 server 接口 `/api/settings/tos-config` 下发。
4. 入库后 `assets.source_url` 应是可公网访问的 HTTPS 链接。

## 相关接口

| 接口 | 说明 |
|------|------|
| `GET /api/mobile/phone/status?phone=手机号` | 检查手机号是否已有 online 账号。 |
| `POST /api/mobile/wechat-login` | 小程序 `wx.login` code 换 token。 |
| `POST /api/mobile/devices/bind` | 使用微信手机号授权 code 绑定手机号账号与手机设备。 |
| `GET /api/mobile/devices` | 查看当前账号的 online 设备与手机设备。 |
| `POST /api/mobile/devices/{device_id}/heartbeat` | 刷新手机设备在线状态。 |
| `GET /api/mobile/downloads?device_id=...` | 获取素材下载列表。 |

## 测试要点

- 手机号未注册时，绑定必须失败并提示没有 online 版本。
- 已注册手机号授权后，再次进入小程序应直接识别为已绑定。
- 素材列表至少覆盖 `assets.source_url`、定时任务结果、H5 会话结果中的视频/图片链接。
- 微信开发者工具里 request/uploadFile/downloadFile 域名校验要打开；若下载失败，优先检查合法域名和链接是否 HTTPS。
