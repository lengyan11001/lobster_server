# 2026-05-13 并发、小程序与素材下载变更纪要

本文记录 2026-05-13 当天已经落地的服务端并发改造、小程序开发进度，以及素材保存/下载的新方式。后续排查或交接时，优先按本文索引到对应文档和代码。

## 一、支持并发的服务端改动

### 已完成

1. 生产库已从 SQLite 切到 PostgreSQL，避免 SQLite 单写锁成为并发写入瓶颈。
2. `backend.run` 支持多 worker：
   - 环境变量：`BACKEND_WORKERS`，兼容 `WEB_CONCURRENCY`。
   - 生产 systemd 当前配置为 `BACKEND_WORKERS=2`。
   - 当前服务器是 2 核 3.6GiB 内存，2 worker 是起步值；不建议直接升 4 worker。
3. Web API 和后台循环任务拆分：
   - `lobster-backend.service` 只跑 Web API。
   - `lobster-background.service` 单进程跑后台循环。
   - 避免多 worker 后每个 worker 都重复执行定时探测、对账、定时发布。
4. MCP 独立为 `lobster-mcp.service`：
   - systemd 下 backend 设置 `LOBSTER_BACKEND_AUTOSTART_MCP=0`。
   - 避免 backend 父进程和 MCP service 抢启动同一个 8001 端口。
5. 数据库连接池默认值已按多 worker 下调：
   - `DB_POOL_SIZE=5`
   - `DB_MAX_OVERFLOW=10`
   - 生产 `.env` 当前没有覆盖这两个值，因此走代码默认值。
6. FastAPI 启动建表、迁移、能力目录同步增加 PostgreSQL advisory lock：
   - lock key：`510051001`
   - 目的：多 worker 同时启动时串行化启动期 DB 结构/目录初始化。

### 后台 worker 负责的循环

代码入口：`backend/background_worker.py`

当前集中在一个单实例进程里启动：

| 任务 | 默认开关 | 说明 |
|------|----------|------|
| `sutui_llm_probe` | `LOBSTER_BACKGROUND_SUTUI_LLM_PROBE_ENABLED=1` | 速推 LLM 定时探测；还受原有实例开关限制。 |
| `sutui_reconcile` | `LOBSTER_BACKGROUND_SUTUI_RECONCILE_ENABLED=1` | 速推对账。 |
| `meta_social_schedule` | `LOBSTER_BACKGROUND_META_SOCIAL_ENABLED=1` | Meta Social 定时发布；需要 Meta 配置存在。 |

生产当前实际启动的是 `sutui_reconcile`，LLM 探测和 Meta 定时发布按现有配置未启用。

## 三、素材保存/下载的新方式

### 背景

H5 在微信内和 iOS Safari 里无法稳定做到“直接保存视频进相册”：

- 微信内打开普通 H5，浏览器权限受限，下载体验差。
- iOS Safari 下载后文件通常进“文件”App 或下载目录，不会自动进相册。
- 浏览器不能绕过用户授权直接写相册。

所以新的主路径是：H5 继续用于聊天和预览，小程序用于素材列表和保存到相册。

### 生成链路

1. online 生成图片/视频/音频。
2. online 调用 `save-url` 入库。
3. 当前要求优先保存稳定 HTTPS 公网链接：
   - 优先走火山 TOS。
   - TOS 配置由云端 `custom_configs.json` 维护，通过 server 下发给 online。
   - 服务器接口：`/api/settings/tos-config`
4. 入库后的素材在 `assets.source_url` 中保存公网 URL。

### H5 链路

H5 结果里返回预览/下载链接，服务端使用 `/api/h5-chat/media` 做代理：

- `disposition=inline`：用于预览/打开。
- `disposition=attachment`：用于下载。

微信 H5 下载体验策略：

- 微信内提示复制链接或用外部浏览器打开。
- iOS Safari 不承诺直接入相册，用户可通过系统分享/保存。

### 小程序链路

小程序使用 `GET /api/mobile/downloads?device_id=...` 获取素材，服务端汇总：

1. `assets.source_url`
2. 定时任务完成结果里的媒体链接
3. H5 会话结果里的媒体链接

返回项包含：

| 字段 | 说明 |
|------|------|
| `url` | 原始公网 URL。 |
| `preview_url` | 服务端代理后的预览链接。 |
| `download_url` | 服务端代理后的下载链接。 |
| `media_type` | `image` / `video` / `audio` / `media`。 |
| `title` | 展示标题。 |
| `created_at` | 创建时间。 |

小程序下载保存：

1. `wx.downloadFile` 下载 `download_url`。
2. 图片调用 `wx.saveImageToPhotosAlbum`。
3. 视频调用 `wx.saveVideoToPhotosAlbum`。
4. 音频或未知媒体默认复制链接。
5. 如果用户拒绝相册权限，引导 `wx.openSetting`。
6. 如果保存失败，复制原始链接作为兜底。
